Hello! I’m Alesia and I want to tell you about CSS Grid Layout.

CSS Grid Layout is a new, powerful layout system that allows a web page to easily be divided into columns and rows.

This CSS module defines a two-dimensional grid-based layout system, optimized for user interface design.

In the grid layout model, the children of a grid container can be positioned into arbitrary slots in a predefined flexible or fixed-size layout grid.

The 'modern' CSS Grid Layout specification began as a first public working draft published on 7 April 2011.

TABLE -> FLOAT -> POSITIONING AND INLINE-BLOCK -> BOX MODEL -> CSS flex box -> CSS grid layout

In the beginning, I want to mention some interesting facts about history of Grid Layout. CSS Grid has been through a lot of discussions and implementations since it was first presented to W3C by a Microsoft team in 2012. The difference was that, in this time, the Microsoft team also presented a real implementation of the grid layout. Several people started playing with it and spreading the word about it. Creators of CSS Grid Layout think that grid-based layouts have been a goal of CSS since the beginning. They remember that CSS started as something very simple. CSS has always been used to lay out web pages, but it's never done a very good job of it. First, tables were used, then floats, positioning and inline-block, but all of these methods were essentially hacks and left out a lot of important functionality (vertical centering, for instance). Flexbox helped out, but it's intended for simpler one-dimensional layouts, not complex two-dimensional ones. The aim of my presentation wasn’t telling you about difference between Grid Layout and Flexbox. However, I should mention that Flexbox and Grid actually work very well together.

Groups of properties

CSS Grid defines according to four groups of properties:

  1. The explicit grid
  2. Gaps
  3. Aligning things
  4. The implicit grid

The explicit grid

You can use two properties to make an explicit grid:

  1. grid-template-columns
  2. grid-template-rows
Let’s say you want to make a grid with 4 columns and 3 rows. If you declare the number of rows and columns in your grid, the grid is explicit. grid-template-columns lets you define the number of columns.grid-template-rows lets you define the number of rows.

grid-template-columns create a new column for each length value you add to it. grid-template-rows work the same way.

grid-template-columns create a new column for each length value you add to it. In the grid-template-columns declaration above, we have four 1fr values. This means four columns. grid-template-rows work the same way.
This creates a grid with four columns and three rows.

Positioning items in your grid

You can control the position of items in a grid with two properties:

  1. grid-column
  2. grid-row
These two properties can only be used on a grid item. grid-column lets you choose which column(s) you want to place your grid item

A shorthand for grid-column-start and grid-column-end works this way: grid-column-start / grid-column-end.

You can also use the span keyword to tell CSS Grid how many columns you want your item to take up.

A shorthand for grid-row-start and grid-row-end works this way: grid-row-start / grid-row-end.

grid-row lets you choose which row(s) you want to place your grid item.

Positioning items in named areas

To create a grid area, you should use grid-template-areas on the grid.

You can name parts of your grid if you don’t like counting rows and columns. These named parts are called grid areas.

Some notes on creating grid areas:

  1. You must name every grid area
  2. If you don’t want to name an area, use .
  3. Each group separated by inverted quotes ("row1" "row2") signifies a row
  4. Each value within inverted quotes ("area1 area2") signifies an area
The example below has three grid areas: 1. header on the first row and takes up 4 columns 2. main on the second row and takes up the middle 2 columns 3. footer on the third row and takes up 4 columns

To place items IN a grid area, you use the grid-area property on the grid item. To place items ON a grid-area, you use grid-area.

Gaps

There are three properties to remember:

  1. grid-column-gap
  2. grid-row-gap
  3. grid-gap
When you create a grid, you can create spaces between columns and rows. These spaces are called gaps. grid-column-gap determines the space between columns. grid-row-gap determines the space between rows.
grid-gap is a shorthand for grid-column-gap and grid-row-gap.

There are six properties to align things:

  1. justify-content
  2. align-content
  3. justify-items
  4. align-items
  5. justify-self
  6. align-self
Aligning things. This is where many people get confused.

Two groups of patterns:

  • The first group is justify vs align
  • The second group is content, items, and self
You can see two groups of patterns here:

Justify vs align

Each CSS Grid has two axes:

  1. The inline-axis
  2. The block-axis

Inline-axis reads left to right. Block-axis reads top to bottom

When you justify something, you’re changing the alignment according to the inline-axis. When you align something, you’re changing the alignment according to the block-axis.

So the inline and block axis is:

  • Inline: left to right
  • Block: top to bottom

Content, items, and self

justify-content and align-content lets you align the grid itself to the available space outside of the grid.

You will only need these properties if your grid is smaller than its defined area. (Which is rare).
You can pick from seven values: 1. start: flush grid to the start of the axis 2. end: flushed grid to the end of the axis 3. center: align grid to the center of the axis 4. stretch: grid fills the axis (this is the default value) 5. space-between: spreads whitespace between grid items. No whitespace at the ends 6. space-around: spreads whitespace around each grid item 7. space-evenly: spreads whitespace evenly around all grid items including the ends

justify-items and align-items lets you align grid-items to any available whitespace in their respective cells.

You can pick from the same four values: 1. start: flush item to the start of the axis 2. end: flushed item to the end of the axis 3. center: align item to the center of the axis 4. stretch: fills the axis (this is the default value)

justify-self and align-self does the same thing as justify-items and align-items. The difference is it lets you change the alignment for only one grid-item.

Implicit Grid

Let’s say you created a CSS Grid, but you don’t have enough rows. In this example an explicit grid was created for three items.

But we have six items!

When you don’t have enough space in your explicit grid, CSS Grid will help you create additional grids automatically. By default, it’ll create more rows.

You can adjust the automatically created column(s)
or row(s) with these two properties:

  1. grid-auto-columns
  2. grid-auto-rows
